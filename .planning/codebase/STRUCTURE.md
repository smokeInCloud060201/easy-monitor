# STRUCTURE

## Directory Layout
### Backend Workspace
- **`shared-proto/`**: Contains `.proto` definitions and generates Rust types via `tonic-build` or `prost`. Shared between agent and master.
- **`master-service/`**:
  - `src/api/`: HTTP REST controllers (`axum`).
  - `src/ingress/`: gRPC service implementations receiving data.
  - `src/bus/`: Internal message passing and event management.
  - `src/processors/`: Telemetry data processing logic.
  - `src/storage/`: Database interaction (`sled`). 
  - `src/main.rs`: Service bootstrapping.
- **`node-agent/`**:
  - `src/collector/`, `src/apm/`, `src/dogstatsd/`, `src/logs/`: Different telemetry collection modalities.
  - `src/wal/`: Write-ahead logging implementation for resilience.
  - `src/forwarder/`: Logic to push data upstream.
  - `src/main.rs`: Agent bootstrapping.

### Frontend
- **`dashboard/`**:
  - `src/`: Standard React Vite structure. Contains components, hooks, pages, and API clients.
  - `public/`: Static assets.

### Infrastructure & Operations
- **`certs/`**: TLS certificates for gRPC communication, likely generated by scripts.
- **`docs/`**: Project documentation payload.
- **`generate_certs.sh`**: Helper script to rotate or create TLS certs.
- **`docker-compose.yml`**: Easy local orchestration to run the whole stack.

## Naming Conventions
- Backend: Standard Rust snake_case for modules/variables, PascalCase for types.
- Frontend: Standard React conventions (PascalCase components, kebab-case CSS classes leveraging Tailwind).
- Proto: Standard protocol buffer directory and naming strategies.
