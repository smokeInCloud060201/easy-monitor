# Plan 03-01: Containerize master-service, dashboard, and write compose file

## Objective
Embed the React dashboard inside the master-service Rust binary/container, and orchestrate the platform (Master + ClickHouse) using Docker Compose.

<action>
Modify `master-service/Cargo.toml` to add `tower-http` features `fs` if not present, to serve static files.
Modify `master-service/src/api/mod.rs` to serve the `../dashboard/dist` folder (or just `./dist` in production) at the root `/` path using `tower_http::services::ServeDir`.
Ensure `/api/v1` routes take precedence over the static SPA routes.

<task>
<read_first>
- master-service/Cargo.toml
- dashboard/package.json
</read_first>
<action>
Create `master-service/Dockerfile` (can be at the repository root to access both folders).
This must be a multi-stage Dockerfile:
1. `node:20` stage: copy `dashboard`, run `npm i` and `npm run build`.
2. `rust:1.77` stage: copy `master-service` and `shared-proto`, run `cargo build --release`.
3. `debian:bookworm-slim` (or similar) final stage: copy the compiled `master-service` binary from stage 2, and copy the `dist` folder from stage 1 into a known location (e.g., `/app/dist`).
Expose port 3000.
</action>
<acceptance_criteria>
- `cat Dockerfile` shows multi-stage build `FROM node`, `FROM rust`, and `FROM debian`.
</acceptance_criteria>
</task>
