# Plan 03-02: Containerize node-agent and document usage

## Objective
Package the `node-agent` so users can deploy the telemetry collector to target hosts without compiling Rust, and provide clear usage instructions.

## Tasks

<task>
<read_first>
- node-agent/Cargo.toml
</read_first>
<action>
Create `node-agent/Dockerfile`.
It should be a multi-stage Dockerfile:
1. `rust:1.77` stage: copy `node-agent` and `shared-proto`, run `cargo build --release`.
2. `debian:bookworm-slim` final stage: copy the binary.
Set the entrypoint to the compiled `node-agent` binary.
</action>
<acceptance_criteria>
- `cat node-agent/Dockerfile` contains a valid multi-stage Rust build.
</acceptance_criteria>
</task>

<task>
<read_first>
- README.md
- docker-compose.yml
</read_first>
<action>
Update the repository `README.md` (or create it if it doesn't exist).
Add a "Quick Start" section demonstrating:
1. `docker-compose up -d`
2. Navigating to `http://localhost:3000`
3. Running the agent: `docker run -d --net=host -e MASTER_NODE_URL=http://localhost:3000 easy-monitor-agent` (or similar instructions depending on how the rust agent accepts master config).
</action>
<acceptance_criteria>
- `grep "docker-compose up" README.md` returns a match.
- `grep "docker run" README.md` returns a match.
</acceptance_criteria>
</task>

## Verification
- Run `docker build -t easy-monitor-agent -f node-agent/Dockerfile .` and verify the compilation succeeds.

## Must Haves
- The agent needs network and PID host access to monitor the generic host if possible, so advise `--net=host` and `--pid=host` in the README if appropriate for host-level telemetry.
