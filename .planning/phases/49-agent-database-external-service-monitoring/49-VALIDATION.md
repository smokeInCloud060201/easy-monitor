# Phase 49 Nyquist Validation Strategy

## Goal Verification
The goals are 1. capturing outbound connections structurally and 2. absolutely masking SQL literals.

## Must-Haves
1. **Node.js**: The agent patches `http.request`, `https.request`, global `fetch`, and prototype chains for `pg` and `mysql2`.
2. **Go**: A `DB` execution wrapper enforces span creation, tagging, and contextual trace progression for DB drivers.
3. **Rust**: `DatadogTracingLayer` natively filters and maps `sqlx` emitted tags (`db.statement`) to obfuscated `db.query` meta keys.
4. **Java**: Generated bytecode securely parses JDBC statements and masks PII out of queries.
5. **Obfuscation Verification**: `"SELECT * FROM users WHERE email='karson@example.com' AND age=25"` MUST be parsed as `"SELECT * FROM users WHERE email=? AND age=?"` globally.

## Tool Verification
Compilers (`go build`, `tsc`, `cargo check`) validate syntax layout correctness. Node prototypes require direct structural verification of the `instrumentation.ts` AST structure.
