# Plan 49-02: Rust and Java Outbound Tracing

<objective>
Implement Outbound HTTP tracing and Database SQL obfuscation via Rust native ecosystem extraction and Java ByteBuddy instrumentation.
</objective>

<requirements>
- Rust `DatadogTracingLayer` must detect and mask `db.statement` logs generated concurrently by `sqlx`.
- Java must natively build `HttpAdvice` and `JdbcAdvice` classes capturing HttpURLConnection headers and raw JDBC string invocations.
- SQL inputs heavily stripped of numbers and strings universally.
</requirements>

## Tasks

### 1. Implement Rust `sqlx` Ecosystem Parsing
<read_first>
- `agents/rust/easymonitor-agent/src/lib.rs`
</read_first>
<action>
In `agents/rust/easymonitor-agent/src/lib.rs`:
1. In `on_close` (or `on_new_span`), check if `span.meta` contains `db.statement`.
2. If `db.statement` exists, it represents a standard OpenTelemetry SQL query natively logged by `sqlx`.
3. Apply standard regex `Regex::new(r"(['""]).*?\1|(\b\d+\b)").unwrap()` using `once_cell` or `lazy_static`.
4. Set the native `span.meta["db.query"]` equal to the `replace_all` obfuscated string.
5. Apply similar mapping if `http.url` is found but `http.method` exists (assume Outbound if not root).
</action>
<acceptance_criteria>
- No specific wrappers required. The agent natively harnesses standard external HTTP and SQL integrations perfectly securely.
</acceptance_criteria>

### 2. Implement Java ByteBuddy Interceptors
<read_first>
- `agents/java/src/main/java/com/easymonitor/agent/ServletAdvice.java`
</read_first>
<action>
Create two new aspect hooks:
1. `HttpAdvice.java`: `@Advice.OnMethodEnter` on `java.net.HttpURLConnection.connect()`. Extract `url` and create an outbound child span matching ID contexts into headers (`setRequestProperty`).
2. `JdbcAdvice.java`: `@Advice.OnMethodEnter` for `java.sql.Connection.prepareStatement(String sql)`. Build a thread-weak mapping holding the query.
3. `@Advice.OnMethodExit` for `PreparedStatement.execute()`. Retrieve mapped query string.
4. Execute `sql.replaceAll("(['\"]).*?\\1|(\\b\\d+\\b)", "?")`. Attach as `db.query` tags to `type: "sql"` span.
</action>
<acceptance_criteria>
- ByteBuddy manipulates native Java classes safely. Database parameters never escape the local application sandbox.
</acceptance_criteria>
