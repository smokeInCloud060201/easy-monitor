<objective>
Filter out logs and traces from internal infrastructure services (master-service, node-agent) at the gRPC ingress layer before they enter the event bus. This prevents internal monitoring data from appearing in the dashboard alongside application service data.
</objective>

<must_haves>
- Logs from `master-service` and `node-agent` are NOT stored in ClickHouse
- Traces/spans from `master-service` and `node-agent` are NOT stored in ClickHouse
- Logs from application services (e.g., `auth-service`, `payment-service`) continue to work normally
- Traces from application services continue to work normally
- Metrics are NOT filtered (they use a different key scheme)
</must_haves>

## Tasks

<task id="14.1.1">
<title>Add internal services exclusion list to utils</title>
<read_first>
- master-service/src/utils.rs
</read_first>
<action>
Add to `master-service/src/utils.rs`:

```rust
/// Services that are part of the monitoring infrastructure itself.
/// Logs and traces from these services should be filtered out at ingress.
const INTERNAL_SERVICES: &[&str] = &["master-service", "node-agent"];

/// Check if a service name belongs to internal infrastructure.
pub fn is_internal_service(service: &str) -> bool {
    INTERNAL_SERVICES.contains(&service)
}
```
</action>
<acceptance_criteria>
- `utils.rs` contains `pub fn is_internal_service(`
- Function returns `true` for `"master-service"` and `"node-agent"`
- Function returns `false` for `"auth-service"`, `"payment-service"`, etc.
</acceptance_criteria>
</task>

<title>Filter logs and traces at gRPC ingress</title>
<read_first>

</read_first>
<action>
Modify `master-service/src/ingress/mod.rs`:

1. Add import:
```rust

```

2. In `sync_logs` (line 56-60), filter entries before sending:
```rust
let entries: Vec<_> = req.entries.into_iter()
    .filter(|e| !is_internal_service(&e.service))
    .collect();

if !entries.is_empty() {
    info!("Received {} logs", entries.len());
    let _ = self.tx.send(Event::Logs(entries));
}
```

3. In `sync_traces` (line 74-79), filter spans before sending:
```rust
let spans: Vec<_> = req.spans.into_iter()
    .filter(|s| !is_internal_service(&s.service))
    .collect();

if !spans.is_empty() {
    info!("Received {} spans", spans.len());
    let _ = self.tx.send(Event::Traces(spans));
}
```
</action>
<acceptance_criteria>
- `ingress/mod.rs` imports `is_internal_service`
- `sync_logs` filters out entries with internal service names before sending to event bus
- `sync_traces` filters out spans with internal service names before sending to event bus

</acceptance_criteria>
</task>
