# Plan 1: Backend Authentication Foundation

<objective>
Replace the hardcoded login stub with real credential validation. Create a ClickHouse users table, implement Argon2id password hashing, externalize the JWT secret to an environment variable, add a seed admin account on first boot, and wire up the POST login endpoint with proper token expiry.
</objective>

<must_haves>
- Users table in ClickHouse with Argon2id password hashes
- POST /api/v1/auth/login endpoint that validates credentials
- JWT secret sourced from JWT_SECRET environment variable
- Seed admin account (admin / changeme) created on first boot
- Proper token expiry (8 hours)
- GET /api/v1/auth/me endpoint returning current user info
</must_haves>

<task id="1.2">
<title>Create ClickHouse users table and seed admin</title>

<read_first>
- master-service/src/storage/clickhouse.rs (current schema initialization pattern)
- master-service/src/storage/mod.rs
</read_first>

<action>
1. In `clickhouse.rs`, add a new `CREATE TABLE` statement to `initialize_clickhouse()`:
   ```sql
   CREATE TABLE IF NOT EXISTS easy_monitor_users (
       id String,
       username String,
       password_hash String,
       role String,
       created_at Int64,
       updated_at Int64
   ) ENGINE = ReplacingMergeTree(updated_at)
   ORDER BY (username);
   ```
2. After table creation, check if any users exist:
   ```sql
   SELECT count() FROM easy_monitor_users FINAL
   ```
3. If count is 0, insert seed admin:
   - Generate Argon2id hash of password "changeme"
   - Insert: `{"id":"seed-admin","username":"admin","password_hash":"<hash>","role":"Admin","created_at":<now>,"updated_at":<now>}`
4. Log a WARNING: "Default admin account created with password 'changeme'. Change this immediately!"
5. Add helper functions:
   - `pub async fn find_user_by_username(client: &Client, username: &str) -> anyhow::Result<Option<UserRecord>>`
   - `pub async fn list_users(client: &Client) -> anyhow::Result<Vec<UserRecord>>`
   - `pub async fn create_user(client: &Client, user: &UserRecord) -> anyhow::Result<()>`
   - `pub async fn delete_user(client: &Client, username: &str) -> anyhow::Result<()>`
6. Define `UserRecord` struct: `{ id: String, username: String, password_hash: String, role: String, created_at: i64, updated_at: i64 }`
</action>

<task id="1.3">
<title>Externalize JWT secret and update auth module</title>

<read_first>
- master-service/src/api/auth.rs (current hardcoded secret and login stub)
- master-service/src/api/mod.rs (ApiState and route definitions)
- master-service/src/main.rs (startup flow)
</read_first>

<action>
1. In `main.rs`, read `JWT_SECRET` env var at startup:
   ```rust
   let jwt_secret = std::env::var("JWT_SECRET")
       .expect("JWT_SECRET environment variable must be set");
   if jwt_secret.len() < 32 {
       panic!("JWT_SECRET must be at least 32 characters long");
   }
   ```
   Pass `jwt_secret` to `start_api_gateway()`.

2. In `api/mod.rs`, update `ApiState`:
   ```rust
   pub struct ApiState {
       pub latest_metrics: Arc<DashMap<String, f64>>,
       pub jwt_secret: String,
       pub ch_client: reqwest::Client,
   }
   ```
   Update `start_api_gateway()` signature to accept `jwt_secret: String`.

3. In `auth.rs`, replace `login_stub()` with `login()`:
   ```rust
   #[derive(Deserialize)]
   pub struct LoginRequest {
       pub username: String,
       pub password: String,
   }

   pub async fn login(
       State(state): State<ApiState>,
       Json(body): Json<LoginRequest>,
   ) -> Result<Json<LoginResponse>, StatusCode> {
       // 1. Query ClickHouse for user by username
       // 2. Verify password with argon2
       // 3. Generate JWT with real claims (sub=username, role=user.role, exp=now+8h)
       // 4. Sign with state.jwt_secret
       // 5. Return { token }
   }
   ```

4. Update `require_jwt()` to use `jwt_secret` from request extensions or state.

5. Add `me()` handler:
   ```rust
   pub async fn me(claims: Claims) -> Json<Claims> {
       Json(claims)
   }
   ```
   Extract claims from request extensions (already inserted by require_jwt middleware).

6. In `api/mod.rs`, update routes:
   - Change `.route("/api/v1/login", get(auth::login_stub))` to `.route("/api/v1/auth/login", post(auth::login))`
   - Add `.route("/api/v1/auth/me", get(auth::me))` inside the protected routes

7. Add `docker-compose.yml` environment section for master-service:
   ```yaml
   environment:
     - JWT_SECRET=dev-secret-key-change-in-production-minimum-32-chars
   ```
</action>

# Start services and test:

  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"changeme"}'
# Expected: 200 with {"token":"eyJ..."}

# 2. Invalid login
curl -X POST http://localhost:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"wrong"}'

# 3. Access protected route with token
curl http://localhost:3000/api/v1/auth/me \
  -H "Authorization: Bearer <token>"

# 4. Access without token
curl http://localhost:3000/api/v1/system/metrics
# Expected: 401
