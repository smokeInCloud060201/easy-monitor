# Phase 45: Agent Native Instrumentation Modules - Java
**Objective:** Implement native ByteBuddy class transformers for Java Servlets and HTTP Clients.

## Context
The Java agent skeleton is in place but contains no actual class transformations. We need to implement Advice classes and a `AgentBuilder` pipeline for inbound HTTP requests.

<action>
In `agents/java/src/main/java/com/easymonitor/agent/EasyMonitorAgent.java`:
1. Use `new AgentBuilder.Default()` to construct the ByteBuddy agent pipeline.
2. Define a transformation for `javax.servlet.http.HttpServlet` or `jakarta.servlet.http.HttpServlet` overriding the `service` method.
3. Create an internal `SpanTracker` utility to hold current trace contexts using `ThreadLocal`.
4. In the `@Advice.OnMethodEnter(suppress = Throwable.class)` block, extract `x-easymonitor-trace-id` headers, start a `web.request` span, and store it.
5. In the `@Advice.OnMethodExit(onThrowable = Throwable.class, suppress = Throwable.class)` block, capture status codes, exceptions, finish the span, and enqueue it for native MessagePack ingestion.
</action>
<acceptance_criteria>
- `EasyMonitorAgent.java` contains `new AgentBuilder.Default().type(...).transform(...)`
- Advice classes have `@Advice.OnMethodEnter` and `@Advice.OnMethodExit` with `suppress = Throwable.class`
- Spans accurately capture HTTP servlet parameters.
</acceptance_criteria>
</task>
