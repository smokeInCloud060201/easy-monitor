# Phase 45: Agent Native Instrumentation Modules - Go
**Objective:** Implement drop-in middlewares for standard Go HTTP clients and servers.

## Context
The Go agent currently provides a basic `WrapHTTPHandler`. It requires a `WrapHTTPClient` for outbound requests.

## Tasks

<task>
<id>45-03-01</id>
<name>Implement HTTP Client wrapper</name>
<read_first>
- `agents/go/telemetry.go`
</read_first>
<action>
In `agents/go/telemetry.go`:
1. Create a `RoundTripper` wrapper named `Transport` that implements `http.RoundTripper`.
2. The `RoundTrip(req *http.Request) (*http.Response, error)` method should start a `http.client.request` span.
3. Inject the `x-easymonitor-trace-id` and `x-easymonitor-parent-id` headers from the current context into the outbound request.
4. Capture `http.status_code` and execute `span.Finish()`.
5. Provide a helper `WrapHTTPClient(c *http.Client) *http.Client` to easily wrap a client's transport.
</action>
<acceptance_criteria>
- `telemetry.go` exports a `WrapHTTPClient` function.
- Outbound requests cleanly inject correlation headers using `req.Header.Set()`.
- Spans are correctly closed and sent to the node-agent via `SendSpan`.
</acceptance_criteria>
</task>

- **must_haves**: The Go agent can cleanly wrap incoming standard library HTTP requests and outgoing `http.Client` requests to propagate unified traces.
