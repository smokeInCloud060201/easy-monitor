# Phase 45: Agent Native Instrumentation Modules - Node.js
**Objective:** Implement native hooks for the `http` and `https` modules in Node.js to capture incoming and outgoing requests natively without OpenTelemetry, using `require-in-the-middle`.

## Context
The Node.js agent currently only wraps the global `fetch` API. To support legacy and framework clients (like Express, Axios, etc.), we need to hook deep into Node's `http` and `https` modules.

## Tasks

<task>
<id>45-01-01</id>
<name>Hook into http and https modules</name>
<read_first>
- `agents/node/instrumentation.ts`
</read_first>
<action>
In `agents/node/instrumentation.ts`:
1. Use the imported `Hook` from `require-in-the-middle` to hook into the `http` and `https` modules.
2. Intercept the exported `request` and `get` functions of `http` and `https` modules.
3. For inbound requests (`http.createServer` emit 'request'), start a new active span `http.server.request` and inject trace context (`x-easymonitor-trace-id`, `x-easymonitor-span-id` if present in headers, otherwise create new).
4. For outbound requests (`http.request` / `http.get`), start a new active span `http.client.request` and inject our current trace context into the request headers.
5. Record appropriate span attributes: `http.method`, `http.url`, `http.status_code`, `error` (if `error` event emitted or status code >= 400).
6. End the span on response `end` or `error` events.
7. Wrap in `try/catch` with fallback to original functions to avoid breaking the application on hook failure.
</action>
<acceptance_criteria>
- `agents/node/instrumentation.ts` contains `Hook(['http', 'https'], ...)`
- `agents/node/instrumentation.ts` implements interceptors for outbound `request` and inbound `Server` requests
- Trace context headers (`x-easymonitor-trace-id`) are correctly extracted and injected using native HTTP events.
</acceptance_criteria>
</task>
