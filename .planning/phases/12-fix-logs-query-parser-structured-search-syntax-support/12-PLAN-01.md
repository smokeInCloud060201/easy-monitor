<objective>
Fix the logs search bar so queries like `service:order-service AND message:"test"` correctly filter logs by service AND message. Implement a frontend query parser that decomposes structured search syntax into individual API filter fields, and extend the backend to support additional filter fields (`host`, `source`, `namespace`, `node_name`).
</objective>

<must_haves>
- Query `service:order-service AND message:"test"` returns only logs from order-service containing "test"
- Query `service:order-service` returns only logs from order-service
- Query `level:ERROR` returns only error logs
- Bare text `connection timeout` searches message field
- Quoted values `message:"connection timeout"` work correctly
- Existing dropdown/sidebar filters still work alongside search bar queries
- Histogram and field statistics respect the parsed query filters
</must_haves>

<read_first>

<action>
Create new file `dashboard/src/lib/queryParser.ts` with:

1. Define `ParsedQuery` interface:
```typescript
export interface ParsedQuery {
  service?: string;
  level?: string;
  pod_id?: string;
  trace_id?: string;
  host?: string;
  source?: string;
  namespace?: string;
  node_name?: string;
  keyword?: string; // remaining bare text → message ILIKE
}
```

2. Define `SUPPORTED_FIELDS` constant:
```typescript
const SUPPORTED_FIELDS: Set<string> = new Set([
  'service', 'level', 'pod_id', 'trace_id',
  'host', 'source', 'namespace', 'node_name', 'message',
]);
```

3. Implement `parseLogQuery(input: string): ParsedQuery` function:
   - Tokenize input by splitting on whitespace, but preserving quoted strings
   - For each token:
     - If matches `field:value` or `field:"quoted value"` pattern:
       - If field is in SUPPORTED_FIELDS: set `result[field] = value`
       - If field is `message`: append to `result.keyword`
       - If field is unknown: append `field:value` to `result.keyword`
     - If token is `AND` or `OR`: skip (AND is implicit, OR not supported yet)
     - Otherwise (bare text): append to `result.keyword`
   - Return ParsedQuery with all parsed fields

Key parsing rules:
- Handle both `field:value` and `field:"multi word value"` syntax
- Strip surrounding quotes from values
- Multiple values for same field: last one wins
- Bare text accumulates into keyword (space-separated)
</action>
<acceptance_criteria>
- File `dashboard/src/lib/queryParser.ts` exists
- File contains `export interface ParsedQuery`
- File contains `export function parseLogQuery(`
- Function handles `service:order-service` → `{ service: "order-service" }`
- Function handles `message:"test"` → `{ keyword: "test" }`
- Function handles `service:order-service AND message:"test"` → `{ service: "order-service", keyword: "test" }`
- Function handles bare text `hello world` → `{ keyword: "hello world" }`
- Function handles `level:ERROR service:payment-service timeout` → `{ level: "ERROR", service: "payment-service", keyword: "timeout" }`
