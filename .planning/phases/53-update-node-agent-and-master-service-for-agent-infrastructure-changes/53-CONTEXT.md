# Phase 53: Update node-agent and master-service for agent infrastructure changes - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Updating the Rust backend (`master-service`) and intermediate proxy (`node-agent`) to ingest, process, and store the new telemetry data types (Profiling, DB spans, Topology, enriched metadata) generated by the Phase 45-52 agent upgrades.

</domain>

<decisions>
## Implementation Decisions

### Data Transport Protocol Updates
- Reuse and update existing `shared_proto` Protobufs to adapt to the new changes. Append Profiling and Topology fields to the existing structures to minimize network multiplexing overhead.

### ClickHouse Schema Enhancements
- Create dedicated OLAP tables in ClickHouse for fast analytical queries on `topology` and `profiling` data.

### Topology Aggregation Approach
- Utilize CQRS background generation within `master-service` to dynamically build topology maps from Trace parent-child IDs asynchronously, keeping the agents lightweight.

### Runtime Profiling Data Persistence
- Store raw runtime profiling (flamegraph) data directly in ClickHouse for deep historical analysis.

### Claude's Discretion
- Code-level implementations, parsing logic for the new fields, database indexing strategies, and gRPC payload sizing limits are left to Claude's discretion.

</decisions>
