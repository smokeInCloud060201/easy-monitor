Provide the endpoints necessary for the dashboard to request a specific trace and receive its hierarchical span data.

Modify `master-service/src/api/queries.rs` to implement the `query_traces` endpoint.

Since ClickHouse tracing tables aren't verified yet, implement a fallback mock generator within `query_traces` just like we did for logs.
If the database query returns empty, generate a mock hierarchical trace with 5-10 spans (one root span, and a few child spans where `parent_id` matches the root's `span_id`). Each span should have a `timestamp`, `duration_ms`, `name`, `service`, `span_id`, and `trace_id`.
</action>

</task>
