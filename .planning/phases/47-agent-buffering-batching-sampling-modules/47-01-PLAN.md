<read_first>
- `agents/node/instrumentation.ts`
- `.planning/phases/47-agent-buffering-batching-sampling-modules/47-CONTEXT.md`
</read_first>
<action>
In `agents/node/instrumentation.ts`:
1. Define constants `MAX_BUFFER_SIZE = 1000` and `FLUSH_INTERVAL_MS = 1000`.
2. Introduce a static probability config (e.g., `let sampleRate = 1.0;`).
3. Modify `Span.end()` to calculate a random probability. If `Math.random() > sampleRate` AND `this.error === 0`, drop the trace (return early).

### 2. Implement Go Agent Channel Buffering
<read_first>
- `agents/go/telemetry.go`
</read_first>
<action>
In `agents/go/telemetry.go`:
1. Refactor from spawning `go func()` on every span to using a central background worker.
2. Add a global `var spanQueue chan *Span` initialized with a capacity of `1000` in `Init()`.
3. Modify `SendSpan()` to use a non-blocking channel send `select { case spanQueue <- s: default: /* drop */ }`.
4. Create a `func startBackgroundFlusher()` invoked during `Init()`.
5. The flusher should use a `time.Ticker` set to 1 second and a local `batch [][]DatadogSpan`.
6. Loop over `select { case <-ticker.C: flush(batch); case s := <-spanQueue: batch.append(s); if len(batch)>100 { flush(batch) } }`.
7. Add head-based sampling in `Finish()`: if a global probability fails and `s.Error == 0`, do not call `SendSpan`.
</action>
<acceptance_criteria>
- `agents/go/telemetry.go` contains `spanQueue = make(chan *Span, 1000)`
- `SendSpan` performs a non-blocking send with `default:` case
- Background flusher uses `time.Ticker` and size batching thresholds.
</acceptance_criteria>
