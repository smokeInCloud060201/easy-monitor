</objective>

### 1. Implement Rust Agent MPSC Batching
<read_first>
- `agents/rust/easymonitor-agent/src/lib.rs`
- `.planning/phases/47-agent-buffering-batching-sampling-modules/47-CONTEXT.md`
</read_first>
<action>
In `agents/rust/easymonitor-agent/src/lib.rs`:
1. Rather than spawning a TCP task in `on_close`, set up a `tokio::sync::mpsc::channel(1000)` during `DatadogTracingLayer::new`.
2. `DatadogTracingLayer` must retain the `mpsc::Sender` handle.
3. In `on_close`, perform a `.try_send()` (non-blocking). If it fails (full), intentionally drop the trace (fail-safe).
4. Implement a background Tokio task in `init_telemetry` or `new` that loops with `tokio::time::interval(1 second)`.
5. The loop should select between the interval tick and the `mpsc::Receiver`. Accumulate spans in a `Vec`.
6. If the `Vec` reaches lengths >= 100, trigger an immediate HTTP flush to the Node agent and clear the vector.
7. Integrate a simple random sampling check in `on_close`: if a global `SAMPLE_RATE` (e.g. `1.0`) is not met and `error == 0`, skip sending.
</action>

- `agents/rust/easymonitor-agent/src/lib.rs` uses `tokio::sync::mpsc::channel`
- `on_close` uses `try_send` instead of spawning a new outgoing request task.

</acceptance_criteria>

### 2. Implement Java Agent Hybrid Flushing
<read_first>
- `agents/java/src/main/java/com/easymonitor/agent/trace/DatadogSpanExporter.java`
</read_first>
<action>
In `agents/java/src/main/java/com/easymonitor/agent/trace/DatadogSpanExporter.java`:
1. The `LinkedBlockingQueue(10000)` is already bounded and fail-safe if `offer()` is used instead of `put()`. Ensure `submit()` uses `queue.offer(span)` (it does).
2. Modify the background daemon worker: replace `queue.take()` with `DatadogSpan first = queue.poll(1, TimeUnit.SECONDS);`.
3. If `first` is null, loop `continue;` (allows the thread to stay agile and perform other timer logic if needed, or just sleep). Wait, if `first` is null, we shouldn't send an empty batch. But if there are items, we drain them up to 99 more.
4. Introduce probabilistic sampling in `submit()`. Generate `ThreadLocalRandom.current().nextDouble()`. If `> sampleRate` AND `span.error == 0`, return early.
</action>
<acceptance_criteria>
- `agents/java/src/main/java/com/easymonitor/agent/trace/DatadogSpanExporter.java` uses `poll(1, TimeUnit.SECONDS)` instead of `take()`.
- `submit()` drops traces based on a programmatic sample rate unless an error occurred.
</acceptance_criteria>
