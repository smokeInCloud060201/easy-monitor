# Plan 01-02: Implement Metrics time-series charts component

## Objective
Build the Recharts-based time-series visualizer on the main dashboard view, hooked into the `useTimeRange` context, and ensure the master-service provides the necessary REST endpoint to feed it.

## Tasks

<task>
<read_first>
- master-service/src/api/metrics.rs
- master-service/src/main.rs
</read_first>
<action>
Create or modify `master-service/src/api/metrics.rs` to expose a REST GET endpoint (e.g., via `axum`) at `/api/metrics`.
The endpoint must accept query parameters `from` and `to`.
For MVP, if the db isn't fully queryable yet, have the endpoint return a mocked JSON array of CPU/RAM percentages over time covering the requested window.
Register this route in the main axum router in `master-service/src/main.rs`.
</action>
<acceptance_criteria>
- `grep "/api/metrics" master-service/src/api/metrics.rs` or `master-service/src/main.rs` returns a match.
- The endpoint code successfully parses `from` and `to` query strings.
</acceptance_criteria>
</task>

<task>
<read_first>
- dashboard/src/lib/api.ts
- dashboard/src/components/metrics/TimeSeriesChart.tsx
- dashboard/package.json
</read_first>
<action>
Create `dashboard/src/lib/api.ts` with a function `fetchMetrics(from: string, to: string)` that calls `http://localhost:3000/api/metrics?from=${from}&to=${to}` (assuming backend is on 3000).
Create `dashboard/src/components/metrics/TimeSeriesChart.tsx` using `recharts`.
It should take a `data` prop. Render a `<ResponsiveContainer>` containing an `<AreaChart>` with an `<XAxis dataKey="time" />`, `<YAxis />`, `<Tooltip />`, and two `<Area>` components (one for CPU, one for RAM).
</action>
<acceptance_criteria>
- `grep "fetchMetrics" dashboard/src/lib/api.ts` returns a match.
- `grep "AreaChart" dashboard/src/components/metrics/TimeSeriesChart.tsx` returns a match.
- `grep "ResponsiveContainer" dashboard/src/components/metrics/TimeSeriesChart.tsx` returns a match.
</acceptance_criteria>
</task>

<task>
<read_first>
- dashboard/src/pages/Dashboard.tsx
- dashboard/src/App.tsx
</read_first>
<action>
Create `dashboard/src/pages/Dashboard.tsx`.
Use the `useTimeRange` hook to get `from` and `to`.
Call `fetchMetrics(from, to)` in a `useEffect` to fetch data.
Render the `TimeSeriesChart` component, passing the fetched data.
Update `dashboard/src/App.tsx` to use `<Dashboard />` instead of the dummy `<div>Metrics</div>` for the `/` route.
</action>
<acceptance_criteria>

</task>
