# Plan 2: Modern APM Dashboard — Datadog-style Multi-Section Service Page

<objective>
Completely rebuild the ServiceDetail page as a modern Datadog-style APM monitoring dashboard. The page will feature multiple dedicated sections: Service Health header, Latency Distribution, Requests & Errors time-series, Error Tracking, Endpoints (resources) with sparklines, Dependencies mini-map, and Recent Traces — all with premium dark-theme aesthetics.
</objective>

<title>Add latency distribution and dependencies API functions to api.ts</title>
<read_first>
- dashboard/src/lib/api.ts
</read_first>
<action>
Add types and fetch functions:

```typescript
export interface LatencyBucket {
  range_label: string;
  min_ms: number;
  max_ms: number;
  count: number;
  percentage: number;
}

export interface LatencyDistribution {
  service: string;
  buckets: LatencyBucket[];
  total_requests: number;
  median_ms: number;
  p50_ms: number;
  p90_ms: number;
  p95_ms: number;
  p99_ms: number;
}

export interface ServiceDependency {
  service: string;
  direction: 'upstream' | 'downstream';
  requests: number;
  error_rate: number;
  avg_duration_ms: number;
}

export interface ServiceDependencies {
  service: string;
  upstream: ServiceDependency[];
  downstream: ServiceDependency[];
}

export async function fetchLatencyDistribution(service: string, from?: string): Promise<LatencyDistribution>
export async function fetchServiceDependencies(service: string, from?: string): Promise<ServiceDependencies>
```
</action>
<acceptance_criteria>
- Types and functions for latency distribution and dependencies exist
- Both handle errors gracefully with fallback empty data
</acceptance_criteria>
</task>
