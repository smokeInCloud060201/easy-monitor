# Plan 1: APM Backend Enhancements — Latency Distribution & Service Dependencies API

<objective>
Add two new backend endpoints to power the modern APM service page: a latency histogram distribution (buckets showing request count per latency range) and a service dependencies endpoint (upstream/downstream services for a given service derived from trace data).
</objective>

## Tasks

<task id="1.1">
<title>Add latency distribution endpoint for service detail page</title>
<read_first>
- master-service/src/api/apm.rs
- master-service/src/api/mod.rs
</read_first>
<action>
Add `GET /api/v1/apm/services/:name/latency-distribution` to `apm.rs`:

```rust
#[derive(Serialize)]
pub struct LatencyBucket {
    pub range_label: String,   // "0-10ms", "10-50ms", "50-100ms", etc.
    pub min_ms: f64,
    pub max_ms: f64,
    pub count: u64,
    pub percentage: f64,
}

#[derive(Serialize)]
pub struct LatencyDistributionResponse {
    pub service: String,
    pub buckets: Vec<LatencyBucket>,
    pub total_requests: u64,
    pub median_ms: f64,
    pub p50_ms: f64,
    pub p90_ms: f64,
    pub p95_ms: f64,
    pub p99_ms: f64,
}
```

ClickHouse query using `histogram(10)(duration)` or manual bucketing:
```sql
SELECT 
    countIf(duration < 10) as bucket_0_10,
    countIf(duration >= 10 AND duration < 50) as bucket_10_50,
    countIf(duration >= 50 AND duration < 100) as bucket_50_100,
    countIf(duration >= 100 AND duration < 250) as bucket_100_250,
    countIf(duration >= 250 AND duration < 500) as bucket_250_500,
    countIf(duration >= 500 AND duration < 1000) as bucket_500_1000,
    countIf(duration >= 1000) as bucket_1000_plus,
    count() as total,
    quantile(0.5)(duration) as p50,
    quantile(0.9)(duration) as p90,
    quantile(0.95)(duration) as p95,
    quantile(0.99)(duration) as p99
FROM easy_monitor_traces
WHERE service = '{service}' AND timestamp >= {from_ms}
FORMAT JSON
```

Mock fallback with realistic distributions.

Register route:
```rust
.route("/apm/services/:name/latency-distribution", get(apm::get_latency_distribution))
```
</action>
<acceptance_criteria>
- Endpoint returns latency buckets with counts and percentages
- Includes percentile values (p50, p90, p95, p99)
- Time range filter via `?from=1h`
- Mock fallback with realistic distribution
</acceptance_criteria>
</task>

<task id="1.2">
<title>Add service dependencies endpoint for dependency mini-map</title>
<read_first>
- master-service/src/api/apm.rs (get_service_map for reference)
</read_first>
<action>
Add `GET /api/v1/apm/services/:name/dependencies` to `apm.rs`:

```rust
#[derive(Serialize)]
pub struct ServiceDependency {
    pub service: String,
    pub direction: String,     // "upstream" | "downstream"
    pub requests: f64,
    pub error_rate: f64,
    pub avg_duration_ms: f64,
}

#[derive(Serialize)]
pub struct ServiceDependenciesResponse {
    pub service: String,
    pub upstream: Vec<ServiceDependency>,
    pub downstream: Vec<ServiceDependency>,
}
```

Reuse the same trace JOIN logic from `get_service_map` but filter for a specific service:

Upstream (services that call this service):
```sql
SELECT parent.service as caller, count() as requests, ...
FROM easy_monitor_traces AS child
INNER JOIN easy_monitor_traces AS parent ON ...
WHERE child.service = '{name}' AND parent.service != '{name}'
GROUP BY caller
```

Downstream (services this service calls):
```sql
SELECT child.service as callee, count() as requests, ...  
WHERE parent.service = '{name}' AND child.service != '{name}'
```

Mock fallback.

Register route:
```rust
.route("/apm/services/:name/dependencies", get(apm::get_service_dependencies))
```
</action>
<acceptance_criteria>
- Returns upstream and downstream service lists with metrics
- Each dependency has requests, error_rate, avg_duration
- Mock fallback with 2-3 upstream and 2-3 downstream
</acceptance_criteria>
</task>
