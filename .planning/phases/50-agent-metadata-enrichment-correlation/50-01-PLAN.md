# Plan 50-01: Global Metadata Enrichment

<objective>
Parse `OTEL_RESOURCE_ATTRIBUTES` synchronously upon library initialization and automatically append `env` and `version` tags into all polyglot generated `DatadogSpan` outputs natively.
</objective>

<requirements>
- Node, Go, Rust, and Java agents must read standard environmental metadata formats.
- Tags must fall back to `"production"` and `"unknown"` respectively if omitted by the application host.
</requirements>

## Tasks

### 1. Extensible Environment Metadata
<read_first>
- All agent core initialization files (e.g. `instrumentation.ts`, `telemetry.go`, etc)
</read_first>
<action>
1. Implement configuration state globally parsing `process.env(...)` or equivalent variables.
2. Tokenize by `,` and then `=` to decouple OpenTelemetry resource standards format parameters (`deployment.environment=staging`).

   - `span.meta["env"] = parsedEnv`
   - `span.meta["version"] = parsedVersion`
</action>
