# Phase 46 - Plan 01: Go Agent Trace Generation & Context Propagation

## Objective
Implement native Span lifecycle management (Start, End) and distributed trace header propagation (`x-easymonitor-*`) in the Go agent without relying on OpenTelemetry SDKs.

## Tasks

```xml
<task>
  <description>Implement Go Span Lifecycle and ID Generation</description>
  <read_first>
    - agents/go/telemetry.go
    - .planning/phases/46-agent-trace-generation-context-propagation/46-RESEARCH.md
  </read_first>
  <action>
    Modify `agents/go/telemetry.go`:
    1. Import `crypto/rand`, `encoding/binary`, `strconv`, `context`, `net/http`, `time`.
    2. Add a helper func `generateID() uint64` that safely generates a random 64-bit unsigned integer using `rand.Read` or `math/rand` seeded properly.
    3. Modify `StartSpan(ctx context.Context, name, resource string) (context.Context, *Span)`:
       - Generate `SpanID` using `generateID()`.
       - Extract existing Span from `ctx` (via `ctx.Value`).
       - If parent exists: set `TraceID = parent.TraceID` and `ParentID = parent.SpanID`.
       - If no parent: set `TraceID = generateID()` and `ParentID = 0`.
       - Set `Start = time.Now().UnixNano()`.
       - Store the new `*Span` into the returned `context.Context`.
    4. Implement `(s *Span) End()`:
       - Set `s.Duration = time.Now().UnixNano() - s.Start`.
       - Marshal and send the span directly via the existing MessagePack / WAL buffered channel (buffer/transport implemented in previous/future phases, but emit it here). Wait, for now just ensure `.Duration` is set properly. We can use a global or package-level `SpanReporter` func or channel.
  </action>
  <acceptance_criteria>
    - `agents/go/telemetry.go` contains `func generateID() uint64`
    - `agents/go/telemetry.go` contains `StartSpan(...) (context.Context, *Span)` with `ParentID` assignment.
    - `agents/go/telemetry.go` contains `func (s *Span) End()` calculating duration via `UnixNano()`.
  </acceptance_criteria>
</task>

<task>
  <description>Implement Go Context Propagation (HTTP/Transport)</description>
  <read_first>
    - agents/go/telemetry.go
    - .planning/phases/46-agent-trace-generation-context-propagation/46-RESEARCH.md
  </read_first>
  <action>
    Modify `agents/go/telemetry.go`:
    1. Define header constants: 
       - `HeaderTraceID = "x-easymonitor-trace-id"`
       - `HeaderParentID = "x-easymonitor-parent-id"`
       - `HeaderSamplingPriority = "x-easymonitor-sampling-priority"`
    2. Implement an HTTP Middleware / Handler wrapper `TraceMiddleware(next http.Handler) http.Handler`:
       - Extract `x-easymonitor-trace-id` and `x-easymonitor-parent-id` from `req.Header`. Handle `strconv.ParseUint(val, 10, 64)`.
       - If trace ID exists, create a dummy root span or inject it into `ctx` so `StartSpan` treats it as parent. Actually, if trace ID exists, create a new span representing the server request, with `TraceID` = extracted trace ID, and `ParentID` = extracted parent ID.
       - Put span in `req.Context()`.
       - Ensure `span.End()` is called natively on return via `defer`.
    3. Implement an outbound `http.RoundTripper` wrapper `Transport struct { wrapped http.RoundTripper }`:
       - In `RoundTrip(req *http.Request)`, extract the active `*Span` from `req.Context()`.
       - If span exists, mutate `req.Header.Set("x-easymonitor-trace-id", strconv.FormatUint(span.TraceID, 10))` and `x-easymonitor-parent-id` = `SpanID` (the current span's ID).
       - Call `wrapped.RoundTrip(req)`.
  </action>
  <acceptance_criteria>
    - `agents/go/telemetry.go` contains `HeaderTraceID = "x-easymonitor-trace-id"`
    - `agents/go/telemetry.go` contains `TraceMiddleware` extracting headers and using `strconv.ParseUint`.
    - `agents/go/telemetry.go` contains a custom `RoundTripper` injecting headers via `strconv.FormatUint`.
  </acceptance_criteria>
</task>
```
