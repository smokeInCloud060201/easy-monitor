# Phase 46 - Plan 03: Node Agent Trace Generation & Context Propagation

## Objective
Implement native Span lifecycle management (Start, End) and distributed trace header propagation (`x-easymonitor-*`) in the Node.js agent using `async_hooks` and standard library monkey-patching.

## Tasks

```xml
<task>
  <description>Implement Node.js Span Lifecycle and ID Generation</description>
  <read_first>
    - agents/node/instrumentation.ts
  </read_first>
  <action>
    Modify `agents/node/instrumentation.ts`:
    1. Import `crypto`. Add helper function `generateId(): bigint` returning `crypto.randomBytes(8).readBigUInt64BE(0)`.
    2. Define `Span` interface containing `trace_id: bigint`, `span_id: bigint`, `parent_id: bigint`, `start: bigint`, `duration: bigint`, etc.
    3. Update the `startSpan(name, resource, callback)` method:
       - Use `AsyncLocalStorage` to get active span context.
       - If active span exists, new span inherits `trace_id`, sets `parent_id = active.span_id`.
       - If no active span, new span generates `trace_id = generateId()`, `parent_id = 0n`.
       - `span_id = generateId()`.
       - `start = process.hrtime.bigint()`. (Or use milliseconds converted to nanos if preferred, but hrtime is better). Actually, use `Date.now() * 1000000n` for wall-clock start time, but use `process.hrtime.bigint()` for duration diffing.
    4. Update `span.end()`:
       - Calculate `duration = process.hrtime.bigint() - start_hrtime`.
       - Queue the span to the buffer (already implemented or just array push).
  </action>
  <acceptance_criteria>
    - `agents/node/instrumentation.ts` uses `readBigUInt64BE` for ID generation.
    - `agents/node/instrumentation.ts` implements `AsyncLocalStorage` based span parenting.
    - Duration is calculated using `process.hrtime.bigint()`.
  </acceptance_criteria>
</task>

<task>
  <description>Implement Node.js Context Ext/Inj (HTTP/HTTPS Patches)</description>
  <read_first>
    - agents/node/instrumentation.ts
  </read_first>
  <action>
    Modify `agents/node/instrumentation.ts`:
    1. Define headers: `X_EASYMONITOR_TRACE_ID = 'x-easymonitor-trace-id'`, `X_EASYMONITOR_PARENT_ID = 'x-easymonitor-parent-id'`.
    2. Patch `http.Server.prototype.emit` (the `'request'` event):
       - Extract headers `req.headers['x-easymonitor-trace-id']` and parse to BigInt.
       - Use `AsyncLocalStorage.run` with a root span whose `trace_id` is the extracted ID and `parent_id` is the extracted parent ID.
    3. Patch `http.request` and `https.request` (outbound):
       - Get active span from `AsyncLocalStorage`.
       - If active span, inject `options.headers['x-easymonitor-trace-id'] = span.trace_id.toString()`.
       - Inject `options.headers['x-easymonitor-parent-id'] = span.span_id.toString()`.
  </action>
  <acceptance_criteria>
    - `agents/node/instrumentation.ts` patches `http.request` to inject `x-easymonitor-trace-id`.
    - `agents/node/instrumentation.ts` patches `http.Server.prototype.emit` to extract `x-easymonitor-trace-id`.
  </acceptance_criteria>
</task>
```
