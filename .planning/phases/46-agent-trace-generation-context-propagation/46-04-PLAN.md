# Phase 46 - Plan 04: Java Agent Trace Generation & Context Propagation

## Objective
Implement native Span lifecycle management (Start, End) and distributed trace header propagation (`x-easymonitor-*`) in the Java agent using ThreadLocals.

## Tasks

```xml
<task>
  <description>Implement Java Span Lifecycle & ThreadLocal Context</description>
  <read_first>
    - agents/java/src/main/java/com/easymonitor/agent/Span.java
    - agents/java/src/main/java/com/easymonitor/agent/ContextManager.java
  </read_first>
  <action>
    1. Create `Span.java`:
       - Define fields: `traceId` (long), `spanId` (long), `parentId` (long), `start` (long nanos), `duration` (long nanos), `name`, `resource`.
       - Implement ID generation: `ThreadLocalRandom.current().nextLong() & Long.MAX_VALUE` (to ensure 64-bit unsigned fits inside signed long as positive, or just parse unsigned everywhere). Datadog uses unsigned 64-bit, so standard 64-bit longs representable via `Long.toUnsignedString()` are fine.
       - Methods: `end()` which calculates `System.nanoTime() - startNanoTime` and pushes to the batcher. (Track absolute wall start time via `System.currentTimeMillis() * 1000_000` but duration via `System.nanoTime()`).
    2. Create `ContextManager.java`:
       - `private static final ThreadLocal<Span> activeSpan = new ThreadLocal<>();`
       - `public static Span startSpan(String name, String resource)`: check `activeSpan.get()`, if exists inherit `traceId` and set `parentId`. If not, generate new root. Set `activeSpan`.
       - `public static void extractAndStart(String traceIdStr, String parentIdStr)`: Parse unsigned longs using `Long.parseUnsignedLong()`, create root span reflecting these, set into `activeSpan`.
  </action>
  <acceptance_criteria>
    - `Span.java` contains `traceId` of type `long`.
    - `Span.java` calculates duration using `System.nanoTime()`.
    - `ContextManager.java` uses `ThreadLocal<Span>` to store active context.
    - Unsigned long parsing uses `Long.parseUnsignedLong()` or similar safety.
  </acceptance_criteria>
</task>

<task>
```
