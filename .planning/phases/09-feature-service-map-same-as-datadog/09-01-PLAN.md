# Plan 1: Service Map Backend — Topology API with Edge Detection from Traces

<objective>
Build backend API endpoints to derive service-to-service dependency edges from trace data and aggregate health metrics per service node. This enables a DataDog-style service map where nodes are services with RED health metrics and edges represent observed inter-service call paths.
</objective>

## Tasks

<task id="1.1">
<title>Create service topology endpoint — derive edges from trace parent-child spans</title>
<read_first>
- master-service/src/api/apm.rs
- master-service/src/api/mod.rs
- master-service/src/storage/clickhouse.rs
</read_first>
<action>
Add a new endpoint `GET /api/v1/apm/service-map` to `apm.rs`:

```rust
#[derive(Serialize)]
pub struct ServiceMapNode {
    pub service: String,
    pub total_requests: f64,
    pub error_rate: f64,
    pub avg_duration_ms: f64,
    pub p95_duration_ms: f64,
    pub status: String,            // "healthy" | "warning" | "error"
    pub node_type: String,         // "service" | "database" | "external" | "cache"
}

#[derive(Serialize)]
pub struct ServiceMapEdge {
    pub source: String,
    pub target: String,
    pub requests: f64,
    pub error_rate: f64,
    pub avg_duration_ms: f64,
}

#[derive(Serialize)]
pub struct ServiceMapResponse {
    pub nodes: Vec<ServiceMapNode>,
    pub edges: Vec<ServiceMapEdge>,
}
```

**Edge derivation logic:**

Query ClickHouse to find inter-service connections from traces where parent span belongs to service A and child span belongs to service B:

```sql
SELECT 
    parent.service as source,
    child.service as target,
    count() as requests,
    countIf(child.error > 0) / count() * 100 as error_rate,
    avg(child.duration) as avg_duration
FROM easy_monitor_traces AS child
INNER JOIN easy_monitor_traces AS parent
    ON child.parent_id = parent.span_id AND child.trace_id = parent.trace_id
WHERE child.service != parent.service
    AND child.timestamp >= {from_ms}
GROUP BY source, target
ORDER BY requests DESC
FORMAT JSON
```

**Node derivation logic:**

For each unique service seen in edges, query RED aggregates:

```sql
SELECT 
    service,
    sum(requests) as total_requests,
    sum(errors) as total_errors,
    avg(duration_avg) as avg_duration,
    max(duration_p95) as p95_duration
FROM easy_monitor_red_metrics
WHERE timestamp >= {from_ms}
GROUP BY service
FORMAT JSON
```

Calculate `status` based on error_rate: >10% = "error", >5% = "warning", else "healthy".
Infer `node_type` from service name patterns (e.g., "redis"/"cache" → "cache", "postgres"/"mysql" → "database", contains "external"/"http" → "external", else "service").

**Mock fallback** returning a sample 5-node topology when ClickHouse is empty:
- api-gateway → payment-service → notification-service
- api-gateway → auth-service
- payment-service → master-service (database)

Add query param support for time range: `?from=1h` (default 1h).

Register in `mod.rs`:
```rust
.route("/apm/service-map", get(apm::get_service_map))
```
