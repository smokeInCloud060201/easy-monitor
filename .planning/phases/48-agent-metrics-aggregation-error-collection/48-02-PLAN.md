# Plan 48-02: Rust and Java Exception String Bounds

<objective>
Update the Rust APM `tracing` layer and the Java ByteBuddy interceptors to extract exception payloads and safely truncate them up to 2000 characters before attaching them to trace metadata maps.
</objective>

<requirements>
- Rust must safely truncate arbitrary strings generated inside `record_str`, `record_error`, and `record_debug`.
- Java must pull the stack from `Throwable.printStackTrace()` and bound the length of the string memory allocation.
</requirements>

## Tasks

### 1. Implement Rust String Alloc Bounds
<read_first>
- `agents/rust/easymonitor-agent/src/lib.rs`
</read_first>
<action>
In `agents/rust/easymonitor-agent/src/lib.rs`:
1. Find `impl tracing::field::Visit for DatadogTracingLayer`.
2. In `record_str`, extract `value.to_string()`. If `> 2000`, truncate it and append `"... (truncated)"`. Insert the clamped value.
3. Keep `record_error` as is, but ensure `value.to_string()` is truncated to 2000.
4. Do the same truncation inside `record_debug`.
5. Apply exactly identical string bounds across the `TagVisitor` variants.
</action>
<acceptance_criteria>
- No arbitrary strings exceeding ~2020 chars can be inserted into the span attributes/metadata map from the tracing hooks.
</acceptance_criteria>

### 2. Implement Java Throwable Stack Serialization
<read_first>
- `agents/java/src/main/java/com/easymonitor/agent/ServletAdvice.java`
</read_first>
<action>
In `agents/java/src/main/java/com/easymonitor/agent/ServletAdvice.java`:
1. Navigate to the `@Advice.OnMethodExit` handler `onExit`.
2. Inside `if (thrown != null)`, extract `thrown.getMessage()` and `thrown.getClass().getName()` safely.
3. Generate the stack trace by creating a new `StringWriter` and `PrintWriter`, and invoking `thrown.printStackTrace(pw)`.
4. Run `toString()` on the `StringWriter`. 
5. If the resulting string length > 2000, slice it `stack.substring(0, 2000) + "... (truncated)"`.
6. Push all three properties to the span (`error.message`, `error.type`, `error.stack`).
</action>
<acceptance_criteria>
- `error.stack` properly traces nested Java throwables and clips appropriately, preventing GC bloat under load.
</acceptance_criteria>
