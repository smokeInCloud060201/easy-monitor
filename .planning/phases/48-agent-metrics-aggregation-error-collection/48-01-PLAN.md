# Plan 48-01: Node and Go Exception String Bounds

<objective>
Update the Node and Go APM middleware telemetry wrappers to extract full exception stacks and safely truncate them up to 2000 characters before attaching them to trace payloads.
</objective>

<requirements>
- Node.js must extract `err.name` and truncate `err.stack`.
- Go must implement `runtime/debug` stack printing on panics and truncate the resulting strings.
</requirements>

## Tasks

### 1. Implement Node.js Error Meta Limits
<read_first>
- `agents/node/instrumentation.ts`
</read_first>
<action>
In `agents/node/instrumentation.ts`:
1. Find `recordException(err: Error)`.
2. Add `this.setAttribute('error.type', err.name || 'Error');`.
3. Read `err.stack` and convert it to a string.
4. If the string length exceeds 2000 characters, truncate it to 2000 characters and append `"... (truncated)"`.
5. Apply the truncated string to `error.stack`.
</action>
<acceptance_criteria>
- `recordException` applies `error.type`.
- The stack trace string is safely bounded to prevent unlimited payload bloating.
</acceptance_criteria>

### 2. Implement Go Panic Stack Buffering
<read_first>
- `agents/go/telemetry.go`
</read_first>
<action>
In `agents/go/telemetry.go`:
1. Add `"runtime/debug"` and `"fmt"` to imports if not present.
2. In `WrapHTTPHandler`, inside the panic `recover()` defer block, generate the stack trace using `stack := debug.Stack()`.
3. Convert the byte array to a string. 
4. Check if the string length is > 2000. If so, slice it to `[:2000]` and append `"... (truncated)"`.
5. Call `span.SetTag("error.type", "panic")`.
6. Call `span.SetTag("error.stack", stackStr)`.
</action>
<acceptance_criteria>
- `runtime/debug` is imported.
- `debug.Stack()` slices are captured and appended to `error.stack`.
</acceptance_criteria>
