# Phase 42: Logs Details UI Improvement - Plan 01

## <objective>
Improve the UI of the log message details. Replace the current "popup" style external detail panel with an inline, expanding accordion behavior where multiple log lines can be expanded simultaneously. Move action buttons to the top right of the expanded view.
</objective>

## <tasks>

<task>
<id>42-01-01</id>
<title>Update Logs State Management</title>
<read_first>
- `dashboard/src/pages/Logs.tsx`
</read_first>
<action>
Modify `dashboard/src/pages/Logs.tsx` state to support multiple expanded logs:
1. Replace `const [selectedLogIndex, setSelectedLogIndex] = useState<number | null>(null);` with `const [expandedLogIndices, setExpandedLogIndices] = useState<Set<number>>(new Set());`.
2. Find all occurrences of `setSelectedLogIndex(null);` (e.g., in `fetchData`) and replace them with `setExpandedLogIndices(new Set());`.
3. Create a new handler function `handleToggleLog`:
   ```typescript
   const handleToggleLog = useCallback((index: number) => {
     setExpandedLogIndices(prev => {
       const next = new Set(prev);
       if (next.has(index)) {
         next.delete(index);
       } else {
         next.add(index);
       }
       return next;
     });
   }, []);
   ```
4. Update the props passed to `LogViewer`: replace `selectedLogIndex` and `onSelectLog` with `expandedLogIndices={expandedLogIndices}` and `onToggleLog={handleToggleLog}`.
</action>
